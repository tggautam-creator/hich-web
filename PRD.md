# HICH — MVP Task List

## How to use this file
Each task below is self-contained. Build the code, write a Vitest test for it, run `npm run lint`, run `npm test -- --run`. Only mark a task `[x]` when all three pass. One task per session.

---

### Week 6 — Payment + Post-Ride + Profile

- [ ] Build `POST /api/rides/:id/end`. Validates QR HMAC. Sets status='completed', ended_at=NOW(). Fare: `fare_cents = max(200, min(4000, Math.round((100 + distance_km*18 + duration_min*5) * 100)))`. `platform_fee_cents = Math.round(fare_cents * 0.15)`. `driver_earns_cents = fare_cents - platform_fee_cents`. In a single database transaction: debit rider wallet_balance, credit driver wallet_balance, insert two transaction records. Returns fare breakdown. Integration test: fare formula, atomic transaction (if credit fails debit rolls back), correct transaction records.
- [ ] Build Wallet at `/wallet`. Shows wallet_balance in dollars (`$${(balance/100).toFixed(2)}`). Transaction history. "Add Funds" navigates to `/wallet/add`. Empty state. Tests for cents-to-dollar display and transaction list.
- [ ] Integrate Stripe. Install `@stripe/react-stripe-js`. Build Add Funds at `/wallet/add`. Amount pills ($10, $20, $50) + custom input (min $5, max $200). Stripe CardElement. Backend: `POST /api/wallet/topup` creates PaymentIntent (test mode). Stripe webhook `POST /api/stripe/webhook` updates wallet_balance on success. Tests: amount validation, insufficient funds state.
- [ ] Build End of Ride Summary at `/ride/summary/:ride_id`. Green checkmark + confetti (plays once). Driver: "You earned $X.XX". Rider: "$X.XX charged". Tappable fare breakdown (base, distance, time, 15% fee, driver net). Route thumbnail. Rate button. Report link. Done → home. Tests for driver vs rider view and fare breakdown expansion.
- [ ] Build Rate Your Ride at `/ride/rate/:ride_id`. 5 tappable stars. 4–5 stars → positive tags. 1–3 stars → issue tags + optional comment textarea. Submit → POST `/api/rides/:id/rate`. - [ ] Build My Profile at `/profile`. Avatar, name, rating. Edit name and phone. Saved routes list from driver_routines (route name, days, time, active/paused toggle, delete). Sign out. Tests for saved routes list and toggle action.
- [ ] Build Emergency Sheet component. Renders in React portal at top of DOM tree on all active ride screens — never inside conditional renders. Backdrop click does NOT dismiss — only an explicit close button. Tests: always in DOM on active ride screens, backdrop does not close, all three options present.
